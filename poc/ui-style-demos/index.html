OpenWeave / UI Style Demo Gallery

Four engineering-first visual directions for the workflow canvas.

Each page uses the same OpenWeave product shell and the same workflow scenario so the comparison stays focused on typography, density, layout, canvas treatment, and operational clarity. These demos are intentionally product-facing rather than artistic.

Graphite Ops

Calm, structured, and highly legible. The safest base for a long-lived engineering control surface.

  • Primer / PatternFly discipline
  • Compact but not cramped
  • Best default for complex operator flows

Blueprint Canvas

More spatial and diagrammatic. Emphasizes the infinite canvas as a system map rather than a dashboard.

  • Architectural drafting cues
  • Sharper canvas identity
  • Best for visual workflow orchestration

Slate Terminal

Low-light, run-heavy, developer-native. Feels closest to an AI execution console.

  • Deep focus environment
  • Strong run and error states
  • Best for agent + terminal workflows

Paper Rail

Quiet and editorial. Treats workflow orchestration as a knowledge work surface with strong document sensibility.

  • Warm, restrained, and calm
  • Content organization first
  • Best for planning-heavy workflows
